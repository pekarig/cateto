Biztonság és adatmentés

A parity védelem NEM helyettesíti a biztonsági mentést! Fontos megérteni a különbséget a hardveres adatvédelem és a valódi backup között, valamint alkalmazni a megfelelő mentési stratégiákat.

Appdata

Appdata Backup plugin

Docker és VM konfigurációk automatikus mentése, amely biztosítja, hogy az alkalmazások beállításai és adatai védve legyenek.

  • Automatikus ütemezett mentések
  • Docker konténer konfigurációk
  • VM beállítások mentése
  • Egyszerű visszaállítás

Rendszer

CA Backup / Restore

Teljes rendszerkonfiguráció mentése, amely lehetővé teszi a szerver gyors helyreállítását katasztrófa esetén.

  • Teljes rendszer konfiguráció
  • Plugin beállítások
  • Hálózati konfiguráció
  • Felhasználói beállítások

Cloud

Duplicati / Duplicacy

Titkosított, verziózott backup cloud-ba vagy külső lemezre, amely biztosítja az adatok biztonságát és helyreállíthatóságát.

  • Titkosított mentések
  • Verziózás támogatás
  • Cloud storage integráció
  • Deduplikáció

Sync

rsync / rclone

Automatizált szinkronizálás másik szerverre vagy NAS-ra, amely folyamatos adatvédelmet biztosít.

  • Automatikus szinkronizálás
  • Távoli szerver támogatás
  • Inkrementális mentés
  • Ütemezett futtatás

Stratégia

3-2-1 szabály

Az iparági szabvány backup stratégia, amely maximális adatbiztonságot nyújt különböző katasztrófa-forgatókönyvek esetén.

  • 3 másolat az adatokról
  • 2 különböző adathordozón
  • 1 offsite (távoli helyszínen)
  • Védelem ransomware ellen

Fontos

Parity ≠ Backup

A parity védelem csak a lemezhibák ellen véd, de NEM helyettesíti a biztonsági mentést. Törlés, ransomware vagy tűz ellen nem nyújt védelmet.

⚠️ Figyelem: A parity csak hardverhiba esetén véd. Mindig készíts külön backup-ot a fontos adatokról!
